# specs/ - 要件・仕様定義

## 担当SubAgent: spec-planner

### 責務
- 目的・スコープ・KGI/KPI を明確化
- 機能要件 / 非機能要件（運用・監査・セキュリティ）を定義
- ITSM / ISO20000 / ISO27001 観点を必ず反映

### 成果物
- `overview.md` - プロジェクト概要・スコープ定義
- `requirements.md` - 機能・非機能要件定義
- `*.spec.md` - 個別機能仕様書

### 既存ドキュメント参照
- `Docs/01_要件定義/` - 詳細な要件定義ドキュメント

### ファイル命名規則
```
YYYYMMDD_feature-name.spec.md
```

### 品質基準
- ITSM / ISO20000 準拠
- NIST CSF 2.0 準拠
- 監査対応可能な証跡性
